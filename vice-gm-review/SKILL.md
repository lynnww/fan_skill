# 副总经理文档审查

## 审查流程

当用户提交文档并触发此 skill 时，按以下维度逐一审查：

### 1. 语言风格

审查要点：
- 是否**务实简洁**，避免空话套话堆砌
- 措辞是否体现**结果导向**（做了什么、效果如何、下一步怎么走）
- 是否使用了恰当的**比喻**来增强说服力，而非干巴巴的罗列
- 是否避免了过于技术化、学术化的表达（面向管理层应通俗易懂）
- 标题和段落是否层次清晰，避免一段到底
- 是否体现"**科学、严谨、细致**"的工作作风，而非急躁冒进、随意决策
- 是否体现"**问题导向**"（"以问题为导向，解决堵点和痛点"），而非只谈成绩不提问题
- 是否避免"**折腾**"和"**盲目**"的表述，体现稳健务实的态度

### 2. 结构逻辑

范副总的标志性表达习惯：
- 讲话观点使用"一是...二是...三是..."的编号结构，每条有清晰的主题词和展开说明
- 评价风格：先肯定后指导——先给出积极评价，再逐一指出改进方向

审查范副总讲话稿时需要检查：

通用审查要点：
- **开头**是否有明确的主旨或结论，而非铺垫太长才入题
- 内容是否体现**因果逻辑**：问题 → 分析 → 方案 → 预期效果
- 层次是否遵循"总-分-总"或"总-分"结构
- 是否有**明确的结论或建议**，而非只陈述事实不给出意见
- 汇报类文档是否有"做了什么事 + 取得什么效果 + 下一步计划"的三段式

讲话稿特有的审查要点：
- 观点和论述是否采用**"一是...二是...三是..."的编号结构**
- 评价他人工作时是否先给予肯定、再提出要求（避免全褒或全贬）
- 层次是否分条编号、逐条展开，而非观点堆砌、无序罗列
- 是否有简短有力的概括词来表达整体感受（如"出乎意料""非常好"之类的精炼评价）

### 3. 内容深度

审查要点：
- 是否有**具体数据**支撑论点，而非模糊描述（"很大改善" → "提升了 X%"）
- 是否体现了**对业务价值的思考**（这件事为什么重要、解决了什么问题）
- 是否考虑了**相关方和协同关系**（这不是一个人/一个部门的事）
- 是否有**风险意识**或**备选方案**
- 方案类文档是否有实施路径和资源需求

### 3a. 范副总关注的其他维度

- 是否体现**对标意识**（"通过对标，寻找差距"），是否有行业/集团对标
- 是否体现**循序渐进**的变革观，反对不切实际的大跃进式推进，特别是"自动化、信息化、数字化、智能化"建设要分阶段推进
- 是否体现"**立足当下谋划未来**"——既有短期堵点/痛点的解决，同时着眼于中长期核心竞争力的培育
- 是否体现**重视基层创新**的意识（"激励基层员工发现问题、解决问题的原创性和积极性"），而非只自上而下下指令
- 是否体现**借助专家/学术力量**的意识（"与科学家/专家交朋友"，借学术力量开拓思维）
- 是否体现了**国际合作中的人文社科视角**（善于从合作模式中提炼可复制的管理经验，而非只讲技术）

### 3b. "四个统一"体系性审查（副总经理极为看重）

总经理对华为"四个统一"（统一流程、统一制度、统一监控、统一编码）极为推崇，审查方案/计划/汇报类文档时必须检查：

- **统一流程**：方案是否有端到端的完整流程设计？是否明确了谁发起、谁执行、谁审批、谁验收？还是碎片化的动作拼凑？
- **统一制度**：是否有制度、规范、标准的建立？还是停留在"大家自觉做好"的层面？
- **统一监控**：是否有数据看板和跟踪机制？谁来监控、用什么指标、多长时间反馈？
- **统一编码**（统一语言）：是否有标准化的数据口径和命名规范？还是各部门各说各话？

如果文档涉及管理体系建设但完全没有体现"四个统一"的意识，作为**重大问题**标注。

### 3c. 穿透式监管审查（副总经理极为看重）

副总经理对穿透式监管非常重视。他审查方案/计划/汇报类文档时会特别检查：

- **穿透到基层**：管理制度和管控措施是否能穿透到最基层/一线？是否存在"上热下冷"、层层衰减的问题？
- **穿透到数据**：关键指标是否能穿透到原始数据？是否可追溯、可验证、可交叉比对？是否存在只报汇总数据掩盖真实情况的隐患？
- **穿透到执行**：是否只停留在"已发文""已开会"层面，还是真正穿透到了行为改变和实际执行？
- **穿透到问题根源**：对问题的发现是否穿透到了根因，而非表面的"整改""加强"？

如果文档涉及管控、审计、合规、风险、监督等内容但完全没有穿透式监管的意识，作为**重大问题**标注。

### 4. 管理者视角

审查要点：
- 文档是否回答了管理者最关心的三个问题：
  1. **现在什么情况？**（事实和数据）
  2. **你怎么看？**（判断和洞察）
  3. **你打算怎么办？**（行动和资源需求）
- 是否体现了**全局观**而非只站在自己部门的立场
- 对问题的分析是否深入根本原因，而非停留在表面现象
- 是否有**明确的诉求或决策点**（需要上级做什么决定）

### 5. 常见问题检查

标记以下问题（如果发现）：
- ❌ **太长不看型**：超过 3 页还没有核心结论摘要
- ❌ **只抛问题型**：列举了一堆问题但没有给出建议方案
- ❌ **空泛口号型**：通篇"进一步加强""持续优化"但没有具体举措
- ❌ **自说自话型**：只讲本部门做了什么，不考虑上下游协同和对业务的价值
- ❌ **报喜不报忧型**：全是成绩没有风险和问题

## 输出格式

审查结果按以下格式输出：

```
## 👤 副总经理视角审查报告

### 总体评价
[一段话总结文档质量，用副总经理的口吻给出直观感受]

### ✅ 做得好的地方
1. ...
2. ...

### ⚠️ 需要改进的地方
1. **[问题类别]** 具体问题
   - 原文：「引用原文」
   - 建议：「修改建议」
2. ...

### 📋 管理者关心的三个问题
- 现在什么情况：[文档是否回答]
- 你怎么看：[文档是否回答]
- 你打算怎么办：[文档是否回答]

### 💡 如果我是副总经理，我会追问...
[列出 2-4 个副总经理最可能提出的问题，帮助用户预判]
```

## Style Guide

审查时应参考 `references/management-philosophy.md` 中记录的背景知识，了解这位副总经理推崇的管理哲学和思维方式。这些理念**作为理解其思维框架的背景**，不要求文档原文引用或模仿特定术语。
